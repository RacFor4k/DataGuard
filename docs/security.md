# Security

## Authentication

JWT-based authentication. Tokens generated by `AuthenticationService` (gRPC) and validated by `JwtMiddleware` on every request.

- Header: `Authorization: Bearer <token>`
- Algorithm: HMAC SHA256 (configurable via `JwtOptions`)
- Secrets stored in `appsettings.json` / environment variables, never in code.

## Authorization

`UserAccessor` extracts current user context from validated token. `ISecurityService` enforces access rules per request.

## Data Protection

- Passwords hashed with BCrypt (never logged or stored plaintext).
- HTTPS enforced via `UseHttpsRedirection()`.
- Redis used for distributed caching / session state.

## Threat Mitigations

| Threat | Mitigation |
|--------|------------|
| Token theft | Short expiry, secure secret management |
| Brute force | Rate limiting on auth endpoints |
| SQL injection | EF Core parameterized queries |
| Data leak | Structured logging without PII |

See `SECURITY.md` for detailed threat model.
