# Безопасность

## Аутентификация

Аутентификация на основе JWT. Токены генерируются сервисом `AuthenticationService` (gRPC) и валидируются middleware `JwtMiddleware` на каждом запросе.

- Заголовок: `Authorization: Bearer <token>`
- Алгоритм: HMAC SHA256 (настраиваемый через `JwtOptions`)
- Секреты хранятся в `appsettings.json` или переменных окружения, **никогда** не хранятся в коде.

## Авторизация

`UserAccessor` извлекает контекст текущего пользователя из валидированного токена. `ISecurityService` применяет правила доступа для каждого запроса.

## Защита данных

- Пароли хешируются с использованием BCrypt (никогда не логируются и не хранятся в открытом виде).
- HTTPS принудительно применяется через `UseHttpsRedirection()`.
- Redis используется для распределённого кэширования и управления состоянием сессий.

## Меры против угроз

| Угроза               | Мероприятие по смягчению                     |
|----------------------|----------------------------------------------|
| Кража токена        | Короткий срок действия, безопасное управление секретами |
| Брутфорс            | Лимитирование скорости на аутентификационных эндпоинтах |
| SQL-инъекции         | Параметризированные запросы в EF Core         |
| Утечка данных        | Структурированная логика без включения PII   |

Подробная модель угроз описана в файле `SECURITY.md`.
