# Services — Сервисы

## Обзор категории

**Services** — слой бизнес-логики и утилит. Содержит классы, выполняющие операции, не связанные напрямую с UI: загрузка ресурсов, HTTP-запросы, кэширование, работа с файловой системой.

### Структура папки

```
Services/
└── IconLoader.cs  # Сервис загрузки и кэширования SVG-иконок
```

### Роль в архитектуре

```
ViewModel → Services → Внешние ресурсы (файлы, сеть, БД)
```

Сервисы отвечают за:
- Загрузку и обработку ресурсов (иконки, изображения, шрифты)
- HTTP/gRPC взаимодействие с сервером
- Кэширование данных
- Валидацию и преобразование данных
- Работу с локальным хранилищем

### Правила оформления

1. **Один сервис = один файл**
2. **Статические или Singleton** — сервисы переиспользуются
3. **Асинхронность** — все I/O-операции через `async/await`
4. **Кэширование** — где уместно, избегать повторных загрузок
5. **Null-проверки** — валидация входных параметров
